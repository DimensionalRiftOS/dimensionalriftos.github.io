Dimensional Rift OS

Linux Distro for I2P

Dimensional Rift is a Debian (stable)-based Linux distribution designed for secure access to the I2P network and providing strong protection against cyber attacks. It can run in live mode or be installed via Calamares.

About Dimensional Rift

Dimensional Rift is a Debian-based GNU/Linux distribution focused on privacy and security. The system is built entirely around I2P (Invisible Internet Project), a peer-to-peer (P2P) anonymous communication network. I2P uses a technique called garlic routing, a form of multi-layer encryption that makes traffic analysis more difficult while also providing protection against attacks.Dimensional Rift comes with a hardened Firefox variant specifically configured for I2P. This browser can only access the I2P network, with clearnet connections deliberately blocked. When clearnet access is required, a separate Firefox browser included in the system can be used instead.

In addition, Dimensional Rift includes multi-layered hardening settings to defend against cyberattacks. The system also enhances privacy by automatically removing certain usage traces upon shutdown.

FAQ

+

Q: What makes Dimensional Rift different from regular Debian?

A: I2P comes pre-installed and configured with an easy-to-use GUI. It allows you to easily configure I2P ports. It comes with a special Firefox variant for I2P that cannot access clearnet, only .i2p sites. The browser and system are resistant to attacks.

Q: I'm new to Linux, is Dimensional Rift suitable for me?

A: If you're interested in the I2P network, yes. Installation and usage are extremely easy.

Q: Is Dimensional Rift open source?

A: Completely open source. You can access it from GitHub.

Q: What is I2P? How is it different from Tor?

A: I2P (Invisible Internet Project) is a network that provides anonymous communication over the internet. Unlike Tor, it's designed for internal network traffic. I2P allows users to create their own anonymous services ("eepsites") and communicate securely within the network.

Q: Garlic Routing vs Onion Routing

A: I2P uses garlic routing instead of Tor's "onion routing". Garlic routing encrypts multiple messages within a single packet, making traffic analysis and tracking even more difficult, enhancing privacy. While Tor encrypts each message with separate layers, I2P bundles packets together ("garlic cloves"), reducing traceability.

Q: Can I access the regular internet with I2P? Are there exit nodes?

A: I2P is primarily designed for services within its own network - eepsites and I2P messaging. Regular internet (clearnet) access is limited. Yes, theoretically "I2P exit nodes" allow internet access, but this isn't as widespread or secure as Tor's method. Exit nodes are generally limited, and it's always safer to stay within I2P for security/anonymity.

Q: Can my ISP see that I'm using I2P?

A: Yes, they can see you're connected but not what you're doing. Your ISP can detect that you're connected to the I2P network and identify that packets belong to I2P, but they cannot see the content, which sites you visit, or your messages due to I2P's end-to-end encryption and garlic routing.

Q: I'm using Dimensional Rift in a restrictive country, what should I do?

A: Enable Hidden Mode in the I2P Router.

Q: Can I use it together with Tor?

A: Theoretically possible but not recommended. Tor + I2P combination requires complex configuration, can reduce performance, and may complicate I2P's own security philosophy. Using I2P within its own network in Rift is the safest method.

Q: What is Firefox I2P Edition?

A: A Firefox variant customized for the I2P network. It blocks ads and scripts and includes many hardening settings. Clearnet access is completely disabled - the entire browser is forced to use the I2P network. Only Firefox and addon updates can use clearnet.

Q: How do I access the regular internet?

A: Use the Firefox Clearnet browser. This is a separate browser for accessing clearnet.

Q: How do updates work?

A: Updates are regularly obtained from Debian repositories. You'll receive security and package updates seamlessly from Debian and I2P repositories. There are no third-party repos.

Q: Who develops this distribution?

A: Dimensional Rift is a side project of the Cyrethium (formerly Securonis) operating system. The distribution is developed by Cyrethium's developer. More info: https://cyrethium.org

Q: Can I use Live USB?

A: You can, but when you install the system, additional hardening settings and AppArmor become active. Therefore, installing is better. Still, the choice is yours - you can carry it as a live USB.

Q: I have an NVIDIA GPU, can I use it?

A: You can use any NVIDIA card supported by the open source nouveau driver. Generally, it will boot without issues, but if your GPU is very old or very new, there might be problems.

Q: Can I use Dimensional Rift as an I2P server?

A: Yes, use I2P Port Manager to configure your port settings. Then you can host your service.

Installation

+

USB Preparation:

  • Prepare a USB drive that you don't use and contains no important data
  • Download Ventoy from its official website, then install Ventoy by selecting your USB drive
  • Copy the ISO file to the location formatted with Ventoy

Booting:

  1. Insert your USB drive into your device
  2. Restart the device and enter the BIOS menu
  3. Move the USB drive to first priority and save
  4. When booting from USB, select the Dimensional Rift ISO from the Ventoy menu
  5. Start your system

Installing to Disk:

  1. Once the desktop environment loads, you're ready - that's all there is to it
  2. If you want, you can install the system to disk using "Install Dimensional Rift" option
  3. The system will be installed with Calamares - just follow the instructions and make the necessary settings
  4. When Calamares reaches the disk installation screen, you can encrypt your disk if you want

Troubleshooting Boot Issues:

  • Don't be hasty - wait a bit
  • Try adding "nomodeset" to boot parameters
  • Some hardware may not have Linux compatibility - unfortunately there's nothing the developer or distribution can do about this

Usage

+

I2P Router Menu:

Use the I2P Router menu located in the dock to start I2P. The "Start I2P Router" option will start I2P and open Firefox I2P Edition - please wait a moment. To see what the options in the I2P Router Menu do, click the about button.

I2P Port Manager:

If you want I2P to work more efficiently, use the I2P Port Manager tool. Please start I2P first so that I2P Port Manager can automatically detect used ports. Use option 8 to check necessary settings. Use option 3 to auto-configure. You can make manual settings, but be careful - if you don't have technical knowledge, you might create security vulnerabilities.

System Update:

To update the system, use the System Update tool - you don't need to do anything else.

Anti-Forensic Settings:

This tool enhances your privacy by deleting logs, RAM, swap, and some other temporary files on shutdown in an installed system. RAM and swap deletion provides protection against cold boot attacks.

Common Issues:

  • Firefox Clearnet shows I2P Edition icon: This is normal. Firefox customizations are done through profiles, so the I2P Edition icon appears each time. But you're using Firefox Clearnet. It's easy to distinguish - if you can't access clearnet, it's I2P Edition.
  • Getting kernel panic: Nothing to worry about - it may be due to hardening settings, just restart.
  • I2P ports opened but still shows "Firewalled": Don't worry if I2P shows "firewalled" - this is mostly normal and I2P still works!
  • Want to use GNOME with Wayland: Tools may have issues since Wayland is incompatible with PyQt5. If you still want to enable it, add # before WaylandEnable=false in /etc/gdm3/daemon.conf
